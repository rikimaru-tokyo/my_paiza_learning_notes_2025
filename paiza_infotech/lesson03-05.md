## 02: 情報に関する理論


### 情報量
ある事象が起こった時にそれがどれぐらい起こりにくいかを
表す尺度のことになります。


情報量:I(E), 生起確率:P(E)
```math
情報量I(E) = \log_2 \frac{1}{P(E)} = -\log_2 P(E)

```

(例題)アルファベットのカード26枚の中からAのカードを引いた時の情報量
```math

情報量I(E) = \log_2 \frac{1}{26} = -(-\log_2 26) = 4.7

```

### 符号化

デジタルデータを別の規則にのっとった異なるデジタルデータに変換すること

#### 1. 通信路符号化

すでに符号化された元のデータに冗長なデータを付加して、さらに符号化する方式付加した、冗長なデータによって通信路で発生した誤りを検出·訂正できる。

```
‘A'→ 符号化 ‘1000001’ + '0’(冗長ビット)
```


#### 2. 情報源符号化
元のデジタルデータを符号化することで送信するデータ量を少なくする



##### ハフマン符号

送信したい情報の中にある出現度の高い値のデジタルデータは短い符号を割り当て、出現度の低い値のデジタルデータには長い符号を割り当ててデータ量を少なくする。

## 04: 文字の表現

![](./images/03/lesson0307.png)

### ASCIIコード

アメリカの規格団体ANSIが制定した文字コードで、アルファベットや数字などを7ビットで表す文字コードです。
アメリカの規格団体が作成したものになるため、日本の文字に関する規定はありません。



### EUCコード(Extended Unix Code:拡張UNIXコード)

UNIXというオペレーティングシステムの国際化対応のために規定された文字コードです。
半角英数字は1バイト、漢字や仮名は2バイトで表現します。


### JISコード

JIS(日本工業企画)で定めた日本語を表現するための文字コードの一つ
英数字や記号などを7ビットまたは8ビットで表現し、平仮名や漢字は16ビットで表現される


### シフトJISコード

シフトJISコードは、JISコードで英数字や記号などを7ビットまたは8ビットで文字を表現していたのに対し、英数字や
記号、漢字をすべて16ビットで表すものになります。


### Unicode

ISOが国際標準として規格化し、全世界共通で使えるよう考え出された文字コード。
他の文字コードとは違い世界中の文字を集め、番号を付けて利用できるようにしているため、Unicodeで管理している文字を実際に使用する際にはエンコーディングというコンピュータ上で利用できる数値に変換するということをしなければいけない。

このエンコーディングの代表的なものにUTF-8、UTF-16、UTF-32があります。



## 05: プログラム言語とコンパイル

### プログラム言語

コンピュータに仕事を指示するプログラムを作成するための言語をプログラム言語といいます。
コンピュータが理解できるのは0と1の二つの状態最初は0と1の組み合わせで記述していた機械語といわれるプログラ
ミング言語を使用していた。
しかし、機械語は、人間には扱いにくい。
そこで、人間に分かりやすく表記できる高水準言語というプログラミング言語が開発された。


### 高水準言語の分類

- 手続型言語
  - コンピュータが実行すべき演算命令やデータ操作命令を順番に記述していくことでプログラムを構成する言語のことを言います
- 関数型言語
  - 関数と呼ばれる一つの処理の塊を入力データによって動作させ戻ってきた値を得る
という、関数の組み合わせで構成される言語です
- 論理型言語
  - 論理式で記述していく言語です
- オブジェクト指向言語
  - データと処理を一体化したオブジェクトとを扱う言語です
- スクリプト言語
  - プログラム言語のうち記述や実行を比較的簡単に行うことができる言語の総称です


### コンパイル

- 高水準言語で書かれたプログラムを翻訳することをコンパイルといいます
- 翻訳される前のプログラムのことを「原始プログラム」または「ソースプログラム」と呼ぶ
- 翻訳後のプログラムのことを「目的プログラム」または「オブジェクトプログラム」と呼ぶ


### コンパイル翻訳手順

1. 字句解析
   - 文字列からなる原始プログラムを字句(トークン)単位に分割する。
2. 構文解析
   - プログラム言語の文法に従って字句構成を解析し、構文木を生成する。
3. 意味解析
   - 構文木の意味を解析し、中間コードを生成する
4. 最適化
   - 実行時の処理効率を高めて実行時間を短縮できるように、中間コードにおける式の簡略化や構成の変更などを行う
5. コード生成
   - 機械語又は中間言語のコードに変換する



## 06: 通信の基本技術 - その1

### 通信回線

情報を伝送する際に使用する物理的な回線のことを通信回線といいます。

- アナログ回線
  - アナログ信号を送信するためのアナログ回線です。
  - 波形信号のことで音声信号などを送る電話回線などが該当します。
- デジタル回線
  - デジタル信号を伝送するためのデジタル回線です。
  - デジタル信号とは0または1で表現される信号のことで、データ通信に向いている回線になります。

### 変調方式

- デジタル信号をアナログ信号に、またはアナログ信号をデジタル信号に変換することを、変調といいます。
- 逆に、変調したデータをもとに戻すことを復調といいます。


### |代表的な変調方式(デジタル信号→アナログ信号)

- AM(振幅変調)
  - デジタル信号の0と1を、アナログ信号の波のあるなしに対応させる方式
  - この方式は、実現が簡単ですが雑音に弱いという欠点がある。
- FM(周波数変調)
  - デジタル信号の0と1を、アナログ信号の低周波と高周波に対応させる方式
  - 振幅変調の次に実現が簡単であり、雑音にも強いという利点がある。
- PM(位相変調)
  - デジタル信号を波形のずれで表現する方式。
  - 波形のずらし方を細かくすることで、一つの波形で複数のビットを送信することができるという利点がある。
- PCM(パルス符号変調)
  - アナログ信号を、一定間隔ごとにサンプリングする標本化。
  - サンプリングした標本値を整数値にまとめる量子化。
  - 量子化した整数値を、2進数で表現することでデジタル化する符号化。
  - この3つの手順によって、アナログ信号をデジタル信号に変調する方式です。


### 信号同期方式

データを送る際、送信側と受信側でタイミングを合わせること

- 代表的な同期方式
  - 調歩同期方式(ビット同期方式)
  - SYN同期方式(キャラクタ同期方式)
  - フラグ同期方式(フレーム同期方式)

### メモリシステム
- レジスタ：実際に処理するデータを記憶

## 02: メモリチップの種類を知ろう

![](./images/05/lesson0504.png)


- RAM(Random Access Memory)
	- 電源を切るとデータが消える = 揮発性
	- データを書き換えできる
- ROM(Read Only Memory)
	- 電源を切ってもデータが消えない = 不揮発性
	- 書き込み回数に制限

- DRAM:ダイナミックRAM
	- リフレッシュが必要
	- 容量あたりのコストが安い
	- 主記憶に一番よく使われている


- SRAM:スタティックRAM
	- リフレッシュが不要
	- 高速で省電力
	- 容量あたりのコストが高い
	- 外付けのキャッシュメモリ、バッテリーバックアップメモリとして利用


- マスクROM
	- 工場で製造時にデータを書き込む
	- 携帯ゲーム機のカートリッジなどに使う


- PROM:プログラマブルROM
	- 1回だけ書き込みできる
	- 書き込み専用装置で書き込む
	- 書き込んだデータは消去できない


- EPROM:イレーザブルプログラマブルROM
	- 繰り返し書き換えできるROM
	- データの消去と書き込みは、専用機器を使う
	- 開発やテストに用いる


- フラッシュメモリ
	- 専用の装置不要
	- 数万回程度、書き換え可能
	- 用途:ファームウエア、SSD

- SDRAM:シンクロナスDRAM
	- システムバスのクロックに合わせて動作

- メモリインターリーブ
	- 複数のメモリモジュールをまたぐようにアドレスを割り当て、
	- 連続したメモリアドレスへのアクセスを同時並行で行う

## 03: 補助記憶装置について知ろう - その１

### 補助記憶装置とは
- SDカード
- USBメモリ
- ハードディスクドライブ(HDD)
- フロッピーディスクドライブ(FDD)
- ソリッドステートドライブ(SSD)
- 光ディスク
- テープドライブ


### 補助記憶装置の特徴
- 電源を切ってもデータが消えない(不揮発性)
- 大容量(数ギガバイト から 数十テラバイト)
- 容量あたりのコストが大幅に安い



### 補助記憶装置の用途
- 仮想記憶：OSが提供する機能で、主記憶からあふれたコードやデータを自動的に入れ替える
- ファイルシステム：OSが提供する機能です。コードやデータを「ファイル」という単位にまとめて、補助記憶装置に格納
- 大容量データの持ち運び(リムーバブル)
- バックアップ：非常時の予備として複製・保存


###  ハードディスクドライブの内部構造
- セクター：同心円状の区域を扇形に区切る
- トラック：同心円状の領域
- シリンダー：複数の円盤にある同じ位置のトラックの集まり


![](./images/05/lesson0507.png)


![](./images/05/lesson0508.png)


### ハードディスクドライブのアクセス時間
- 平均位置決め時間：ヘッダーの移動時間
- 平均回転待ち時間：セクターの移動待ち時間
- 平均待ち時間：平均位置決め時間 ＋ 平均回転待ち時間
- データ転送時間：データを転送する時間
- アクセス時間：平均待ち時間 ＋ データ転送時間

## 07: 入力デバイスについて知っておこう

- 入力装置:外部からデータを入力するデバイス
	- ユーザの操作を取り込む(マウス、キーボード)
	- 外部の信号を取り込む(カメラ、マイク)
	- 情報やデータを読み込む(ICカード、バーコード)

- ポインティングデバイス
	- 画面と組み合わせて利用
	- 画面上のポインタを直接触った位置を取り込む
	- 画面上のメニューやアイコンを選択
		- 例:マウス、トラックボール、タッチパネル、タッチペン


- 文字入力:キーボード
	- 文字を入力
	- タイプライタ状の文字キーを持つ
- ゲームコントローラ、ジョイスティック
- ジェスチャー
- 位置センサー、傾きセンサー

- タッチパネルの方式

- 感圧式·抵抗膜方式
	- 2枚の透明電極膜を貼り付け
	- 押した位置の電圧の変化を検出
- 静電容量方式
	- 指と導電膜の間の静電容量の変化を検出
	- 複数のタッチを検出できる
- 電磁誘導方式
	- 専用のペンで磁力を発生させて、その変化を検出

- 画像入力
	- カメラ:写真や動画を撮影する
	- イメージスキャナー:平面の文書や画像を取り込む
- 音声入力
	- マイク
